NAME

mlle2hbook - konvertiert MLLE-Daten ins HBOOK-Format

AUFRUF

       mlle2hbook [-ivhu]
                  [-I hbid] [-d hbdir] [-t title]
                  [-b binsize] [-p params] [-P -S psizes]
                  [-Q quota] [-R range] [-r count]
                  hbfile [mllefiles]

BESCHREIBUNG

       Mlle2hbook  konvertiert  Daten,  die unter dem MLLE-System
       erzeugt wurden, ins HBOOK-Format (-> cernlib).  Die resul-
       tierenden  HBOOK-Dateien  können  dann mit diversen CERN-
       Programmen (z.B. paw++) weiterverarbeitet werden.
       Mlle2hbook konvertiert sowohl Rohdaten  (rdt-Dateien)  ins
       HBOOK-Ntuple-Format   als  auch  Spektren  (osp-Files)  in
       HBOOK-Histogramme.

ARGUMENTE

       hbfile
               [HBOOK File (Output)]
               Name der HBOOK-Datei,  in  die  die  konvertierten
               Daten  geschrieben  werden  sollen.  Die Extension
               sollte .hbook  sein;  sie  wird  ggf.  automatisch
               ergänzt.
       mllefiles
               [MLLE Files (Input)]
               MLLE-Datei(en),  die  konvertiert werden soll(en).
               Fehlt  dieses  Argument,  so  werden   Daten   von
               /dev/dat    gelesen.    Werden   mehrere   Dateien
               angegeben, wird die HBOOK-Kennung je  Datei  um  1
               erhöht.

OPTIONEN

       -i      [Interactive Mode]
               gibt den File-Header der MLLE-Datei auf stderr aus
               und fragt ab, ob die Datei ins  HBOOK-Format  kon-
               vertiert werden soll.
       -h      [Output Header]
               gibt  den  File-Header  der  MLLE-Datei auf stderr
               aus.
       -v      [Verbose Mode]
               gibt ausführliche  Diagnosemeldungen  auf  stderr
               aus.
               Die  Option  -vv ("very verbose") gibt zusätzlich
               eine Record-Statistik aus, -vvv startet den Debug-
               Modus.
       -u      [Update File]
               schreibt  die  konvertierten Daten in eine bereits
               existierende HBOOK-Datei: Daten  mit  einer  neuen
               Kennung  werden  der  HBOOK-Struktur hinzugefügt.
               Daten  mit  bereits  vorhandener  Kennung   über-
               schreiben den existierenden Datensatz.
               Normalerweise  (d.h.  wenn -u nicht angegeben ist)
               wird hbfile neu erzeugt.
       -I hbid [HBOOK Id]
               definiert die Kennung, unter  der  die  Daten  ins
               HBOOK-Format  übertragen  werden sollen. Der Wert
               hbid=1 ist voreingestellt.
       -d hbdir
               [HBOOK directory]
               legt innerhalb des HBOOK-Ordners  das  Unterdirec-
               tory  hbdir  an  und  speichert den Datensatz dort
               unter der Kennung hbid ab.
       -Q quota
               [Change HBOOK quota]
               setzt die maximale  Grösse  der HBOOK-Datei  auf
               quota*1000   Records. Standardmäßig ist  eine
               HBOOK-Datei auf  32000  Records  (je  1024  Worte)
               beschränkt (d.h. 128 MB).
       -t title
               [HBOOK Title]
               vereinbart  den Titel der HBOOK-Datei. Fehlt diese
               Option, so werden als  Titel  der  Kommentar,  das
               Datum  und die Uhrzeit der MLLE-Datei übernommen.
       -p params
               [Param Names]
               definiert die Namen der Event-Parameter.  Es  gibt
               zwei Möglichkeiten, die Namen anzugeben:
               (1)  durch eine Zeichenkette der Form p1:p2:...:pN
               oder
               (2) durch Angabe einer Datei mit Extension .pn.
               Fehlt die Option -p, so werden die Parameter fort-
               laufend numeriert: p1, p2, ..., pN.
               Einträge  in  der  .pn-Datei bestehen aus Zeilen,
               die jeweils den Namen eines  Parameters  und  ggf.
               die Auflösung in Kanälen enthalten.
               Die   Option  -p  bietet  auch  die  Möglichkeit,
               einzelne Parameter  auszuwählen,  d.h. nur  eine
               Untermenge   der   MLLE-Parameter   ins  HBOOK  zu
               übernehmen: Parameter, deren Name mit 0 angegeben
               wird,  werden nicht übernommen. Eine Null am Ende
               der Liste setzt alle  weiteren  Parameter  auf  0.
               Dadurch kann die Größe der resultierenden HBOOK-
               Datei verringert werden.
       -S psizes
               [Param Sizes]
               definiert die Auflösung in Kanälen in der  Form
               s1:s2:...:sN.   Sinnvoll  nur für Histogramme und
               Projektionsspektren (Option -P).
       -P      [Projections Only]
               In diesem Fall werden  keine  Ntupel  konvertiert,
               vielmehr  werden  direkt  Projektionen in Form von
               1-dim Histogrammen angelegt. Eignet sich z.B.  zum
               Anlegen von Eichspektren.
       -b binsize
               [Bin Size]
               definiert  die  Binsize  (=  Datenpunkte je Kanal)
               für Histogramme und Projektionen.
       -R range
               [Record Range]
               legt fest, welche  Records  des  Datensatzes  kon-
               vertiert  werden sollen. Die Grenzen werden in der
               Form  lrl:url  (lower  /   upper   record   limit)
               angegeben,  die  Notierung  lrl: bedeutet "Beginne
               Konversion ab  Record  lrl",  die  Notierung  :url
               "Konvertiere  alle  Records  einschließlich url".
               Fehlt die Option -R, so werden sämtliche  Records
               der Datei übernommen.
               Zu  beachten:  Record#1  (der File-Header) wird in
               jedem Fall gelesen.
       -r count
               [Repeat Count]
               konvertiert  automatisch   die   folgenden   count
               Dateien. Die HBOOK-Kennung wird für jede Datei um
               1 erhöht, ausgehend von        Startwert  hbid.   Diese
               Option  setzt  voraus, daß Daten von Band gelesen
               werden.
