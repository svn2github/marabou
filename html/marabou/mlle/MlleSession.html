Beispiel für eine Sitzung an der MLLE (ab Mai 1998)

  1. Login am CADMUS unter dem Gruppenaccount, z.B. gams
  2. Wechsel zum Directory für die Datenaufnahme:
    • cd /o/exp/gams
  3. Überblick über die Disk-Belegung:
    • df /o/exp und u.U.
    • du -s /o/exp/böser_user
  4. Initialisierung der MLLE
    • ms oder
    • msi (falls Setup interaktiv geändert werden soll)
      • Type of MLLE cpu [EUROCOM_RAM]? <CR>
      • Environment (normal, test) [normal]?<CR>
      • Number of processor boards [...]<CR>
      • Number of AIAC processors [...]<CR>
      • Device for data i/o [disk]?<CR> NEU!!
      • Task name for data i/o [disk]?<CR> NEU!!
      • Enter automatic setup mode [y]?<CR>
  5. Start des Online- bzw. Offline-Programms (wie gehabt)
  6. Wegschreiben/Einlesen der Rohdaten
    • Online:
      • rdtfnames format (definiert Generator für Filenamen: normal run%R.rdt)
      • oopen file.rdt bzw.
      • oopen (inkrementiert Filenamen automatisch)
    • Offline:
      • mlle "set RDT_HSIZE nnn" (falls Headersize vom Standard abweicht)
      • iopen file.rdt bzw.
      • iopen (fragt nach dem Filenamen)
      • Filenamen können auch relativ zu einem (oder mehreren) Pfaden eingegeben werden: dazu setze man
        • export INPPATH=pathspec in Unix bzw.
        • set INPPATH pathspec im Offline-Programm
    • ACHTUNG! Das Kommando show rdth darf nicht mehr verwendet werden. Der Fileheader wird jetzt beim iopen-Befehl automatisch ausgegeben. Da beim Lesen von Disk keine Record-Struktur mehr erkennbar ist, würde ein weiteres show rdth die Synchronisation des Dateninputs mit den jeweiligen Blockanfängen zerstören.
  7. Verfügbarer Platz
    • Das Filesystem /o/exp hat momentan eine Kapazität von 27 GB
    • Jede Gruppe kann temporär (d.h. für 14 Tage) bis zu 10 GB belegen
    • ggf. müssen die Rohdaten-Files mit gzip (am Charly) komprimiert werden
  8. Datensicherung
    • Die Sicherung der Rohdaten sollte über das Archivierungssystem des LRZ geschehen
    • Am Charly: dsm
      • Button Archive files into long term storage klicken
      • Der Reihe nach durch den Datei-Baum ("+" anklicken)
        • CHARLY
          • Local Filespaces
            • a/charly/o/exp
              • gams
    • Zu sichernde Datenstrukturen durch Anklicken der Box am Anfang der Zeile aktivieren
    • Button Archive drücken
    • Es wird jedesmal auf dem LRZ-Server ein neuer Datensatz unter Datum und Uhrzeit angelegt (Achtung! Aus Versehen können so identische Rohdatenfiles mehrfach archiviert werden...)
    • Die lokale Datensicherung am BL (z.B. auf DAT am Charly) ist natürlich nach wie vor möglich, wird aber nicht mehr empfohlen.
  9. Konvertierung alter MLLE-Rohdaten von DAT auf Disk
    • Wer die alte Datenwelt der MLLE verlassen möchte, sollte alte DAT-Bänder mit mtcopy auf die Disk übertragen und von da an die Auswertung wie oben beschrieben fortführen
    • Momentan nur auf SGIs (z.B. Charly) möglich:

      mtcopy ifile=/dev/dat ofile=file.rdt format=mlle ohs=364 mode=header,inter

  10. Falls es Probleme gibt .... EMail an die Rechnergruppe
