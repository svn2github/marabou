NAME

mtcopy - copy tapes

SYNTAX

       mtcopy [option=value] [option=value ...]

BESCHREIBUNG

       Mtcopy kopiert Files zwischen zwei Bandgeräten bzw. zwis-
       chen Bandgerät und Disk.  Die Ein-/Ausgabegeräte können
       sich dabei an beliebigen Rechnern im Netz befinden.
       Eine  Ausgabe  der  Daten  auf  TTY  in Klartext (Hex- und
       ASCII-Dump)  ist   ebenfalls   möglich. Mtcopy   weiß
       außerdem  über      das  Rohdatenformat an der MLLE bescheid
       und kopiert die Daten dementsprechend.
       Schließlich kann mtcopy - über Pipes  mit  anderen  Pro-
       grammen verbunden - als Filter eingesetzt werden.

OPTIONEN

       Die  Eingabe für mtcopy erfolgt durch Angabe von Optionen
       im Stil von dd(1).
       Alle Optionen haben die Form
                    option=value.
       Dabei ist option  einer  der  im  folgenden  aufgeführten
       Namen,  value  je  nach  Art der Option eine Zahl oder ein
       ASCII-String.  An eine  Option  können bei  Bedarf  auch
       mehrere  Werte value übergeben werden, jeweils durch Kom-
       mas getrennt:
                    option=val1,val2, ...
       Leerzeichen sind nur zwischen einzelnen Optionen  erlaubt,
       nicht  jedoch  zwischen einer Option und ihrem/n Wert(en).
       Optionen  und  ihre  Werte  können  innerhalb  der   Ein-
       deutigkeit abgekürzt werden.
       Es  folgen  die  in mtcopy erlaubten Optionen zusammen mit
       den möglichen Werten.
       ifile=device
               definiert das Gerät, von dem  die  Daten  gelesen
               werden  sollen.   Fehlt die Angabe ifile=, so wird
               von stdin gelesen.
       ofile=device
               definiert  das   Gerät, auf   das   die   Daten
               geschrieben   werden  sollen.   Fehlt  die  Angabe
               ofile=, so wird auf stdout  geschrieben.   Handelt
               es  sich  beim Ausgabegerät um ein TTY, so werden
               die Daten automatisch in  ASCII  ausgegeben  (sog.
               Hex-/ASCII-Dump, siehe auch oformat=ascii).
               Der Gerätename wird im allgemein üblichen Format
               angegeben:
                    [[user@]host:]/dev/device  (für Bandgeräte)
                    [[user@]host:]path  (für Diskfiles)
               Dabei  ist host der Name eines beliebigen Rechners
               im Netz, user der Name  des  Ziel-Accounts.  Fehlt
               die  Angabe eines Rechnernamens, so wird erwartet,
               daß sich das Gerät am lokalen Rechner      befindet.
               Unterscheidet  sich  user  vom Namen des aktuellen
               Benutzers, müssen Zugriffsrechte (über .rhosts)
               gegeben sein.
               Eine  Reihe  von Gerätenamen ist im Programm fest
               vereinbart: Für sie kann die Angabe des Vorspanns
               /dev/... entfallen:
               /dev/dat       4mm Digital-Audio-Band
               /dev/exabyte   8mm Exabyte-Video-Band
               /dev/streamer  QIC-Streamer-Band
               /dev/tape      normales 1/2-inch Magnetband
       idevice=device
               Synonym für ifile=
       odevice=device
               Synonym für ofile=
       ibsize=bytes
               definiert  die  Größe eines Blocks/Records für
               das Eingabegerät.
               Für Bandgeräte muß die Record-Größe angegeben
               werden,  für  Diskfiles bzw.  Eingabe über Pipe
               (stdin)  ist  eine   Größe   von   10240   bytes
               (=20x1024) voreingestellt.
       obsize=bytes
               definiert  die  Größe eines Blocks/Records für
               das Ausgabegerät.
               Wird die Record-Größe      für  die  Ausgabe  nicht
               (oder  mit  0)  definiert,  so  richtet  sie  sich
               aktuell nach der Größe des  jeweiligen      Eingabe-
               Records.  Auf diese Weise lassen sich bequem Disk-
               files übertragen, die  ja  im  allgemeinen  keine
               feste Record-Struktur besitzen.
               Weichen  die  Angaben  für  ibsize=  und  obsize=
               voneinander ab, so werden die Daten beim  Kopieren
               entweder    abgeschnitten    oder    mit    Nullen
               aufgefüllt.
       bsize=bytes
               definiert die  Record-Größe  für  Eingabe-  und
               Ausgabegerät   gemeinsam,   entspricht also  den
               Befehlen ibsize=bytes und obsize=bytes.
       ihsize=bytes
               definiert die Größe des  Header-Records  (Record
               #1 eines Datensatzes) auf der Eingabeseite.
       ohsize=bytes
               ist die entsprechende Definition für die Ausgabe-
               seite.
       hsize=bytes
               vereinbart gleiche  Header-Größen  für  Eingabe
               und Ausgabe.
               Diese   Optionen   sind  nur  sinnvoll,  wenn  die
               Datensätze mit speziellen Header-Records beginnen
               (siehe auch format=header). Möglicherweise weicht
               dann die Größe des Headers von der der      übrigen
               (Daten-)Records  ab  (wie  z.B.  im Fall des MLLE-
               Formats).  Fehlt  die  Spezifikation  der  Header-
               Größe, so wird hsize=bsize angenommen.
       iformat=fmt
               definiert das Eingabeformat.
       oformat=fmt
               definiert das Ausgabeformat.
       format=fmt
               vereinbart  dasselbe  Format  auf der Eingabe- und
               auf der Ausgabeseite.
               Die möglichen Formate sind weiter unten unter der
               Rubrik  FORMATE beschrieben. Mehrere Formatangaben
               können - durch Kommas getrennt - innerhalb  eines
               format-Befehls gemacht werden.
       conv=fmt
               vereinbart  die  Art  der  Datenkonversion. Dieser
               Befehl  wurde  aus  Kompatibilitätsgründen   mit
               dd(1)  eingeführt  (in der Form conv=swab - swap
               bytes on input). Man verwende besser iformat=...
       repeat=count
               wiederholt den gesamten  mtcopy-Befehl  count-mal.
               Damit  lassen  sich  ganze  Bänder kopieren. Nor-
               malerweise wird eine mtcopy-Anweisung  nur  einmal
               ausgeführt. Erfolgt die Ausgabe auf Disk, so wird
               an  den  Filenamen  jeweils  die  laufende  Nummer
               angehängt: ofile-n.
       files=count
               faßt  count  Eingabe-Dateien in eine Ausgabedatei
               zusammen (concatenation).  Besitzen  die  Eingabe-
               Dateien  Header-Records,  so  wird  -  falls nicht
               anders definiert -  jeweils  nur  der  Header  der
               ersten  Datei  auf die Ausgabeseite kopiert (siehe
               auch oformat=subheader).  Normalerweise werden die
               Dateien  1:1  von  der  Eingabe  auf  die  Ausgabe
               kopiert.
       skip=nrecs
               spult   das   Eingabegerät   vor    Beginn    der
               Datenübertragung  um  nrecs  Records  nach  vorne
               (normalerweise skip=0).
       count=nrecs
               Transferiert nur nrecs Records (normalerweise alle
               Records einer Datei).
               Die  Befehle skip= und count= entsprechen denen in
               dd(1).   Eine  weitaus  flexiblere   Möglichkeit,
               einzelne  Eingabe-Records  für den Kopiervorgang
               auszuwählen, bietet der range-Befehl.
       range=rng
               definiert die Records, die kopiert werden  sollen.
               Das  Argument  rng besteht aus einer oder mehrerer
               Angaben der Form
                    start[:end],
               die jeweils durch Kommas voneinander getrennt sein
               müssen.        Die Eingabe von end=0 bedeutet dabei bis
               End-of-File.  Fehlt die Angabe von  end,  so  wird
               start=end gesetzt, d.h. nur 1 Record transferiert.
               Die  dd-Schreibweise  skip=n  count=m  kann  durch
               einen  Befehl der Form range=n:n+m ersetzt werden.
       mode=mod
               vereinbart die Betriebsart des Programms.  Mehrere
               Werte   für   mod  können  mit      Kommas  aneinan-
               dergehängt werden. Die        möglichen  Einstellungen
               sind weiter unten (BETRIEBSARTEN) aufgeführt.
       help=[xyz]
               druckt  einen  erklärenden  Text für das Syntax-
               Element xyz (eine Option, ein Format,  ein  Gerät
               oder  eine  Betriebsart). help= ohne Argument gibt
               eine Liste aller möglichen  Kommandos  und  ihrer
               Werte aus.  Die Ausgabe erfolgt auf stderr.
       usage=[xyz]
               ist ein Synonym für help=.

FORMATE

       Folgende   Formate   können  in den  Befehlen  iformat=,
       oformat= und format= angegeben werden:

       mlledata
               (Eingabe und Ausgabe)
               gibt an, daß es sich bei den  Daten  um        Rohdaten
               von der MLLE handelt.  In diesem Fall wird automa-
               tisch eine Record-Größe von bsize=10240 angenom-
               men,  die  Einstellung format=header wird gesetzt,
               und die Größe eines Header-Records mit hsize=512
               vereinbart.
               Anhand des Kennworts im Header der MLLE-Daten wird
               erkannt, ob es sich um das richtige Byte  Ordering
               handelt;  ggf.  wird  automatisch  auf  swap bytes
               (iformat=swapbytes) umgeschaltet.
       header  (Eingabe)
               gibt an, daß die Eingabe-Daten mit einem  Header-
               Record  (Record  #1)  mit  besonderer Struktur und
               u.U. abweichender Länge beginnen.
               (Ausgabe)
               gibt an, daß die Header-Records von der        Eingabe-
               seite  auf  die  Ausgabeseite  übernommen  werden
               sollen.  Definiert man  nur  iformat=header,  aber
               nicht  oformat=header,  so  werden die Daten unter
               Weglassen der Header-Records kopiert.
       subheader
               (nur Ausgabe)
               gibt  an,  daß        auch   sämtliche   nachfolgenden
               Header-Records kopiert werden sollen. Diese Option
               ist dann sinnvoll, wenn mehrere Eingabedateien  zu
               einer  Ausgabedatei  zusammengefaßt werden sollen
               (concatenation, files>0): Im Normalfall wird  dann
               nur  der  Header  der ersten Datei übernommen, es
               sei denn, man setzt oformat=subheader.
       swapbytes
               (nur Eingabe)
               definiert, daß die Datenbytes nach  dem        Einlesen
               paarweise    vertauscht    werden   sollen   (big-
               endian/little-endian-Problem). Im Fall  von  MLLE-
               Daten  wird  die Wahl des  richtigen Byte Ordering
               durch   ein   Kennwort   zu   Beginn   der   Daten
               sichergestellt.
       swab
               ein Synonym für swapbytes (dd-Kompatibilität).
       ascii   (nur Ausgabe)
               gibt  die  Daten  im ASCII-Format aus. Es wird das
               übliche Ausgabe-Format        mit  einer  Mischung  aus
               hexadezimaler und ASCII-Darstellung verwendet.
               Diese  Option  wird  automatisch  gesetzt, wenn es
               sich beim Ausgabegerät um ein TTY handelt.
               Im Fall von MLLE-Daten werden Header-  und  Daten-
               Records  speziell  dekodiert  und im Klartext aus-
               gegeben.

BETRIEBSARTEN

       Der Befehl mode= kennt folgende Werte:

       verbose gibt ausführliche Status- und  Diagnose-Meldungen
               auf stderr aus.
       interactive
               fragt  nach, ob eine Datei wirklich kopiert werden
               soll.  Sinnvollerweise setzt man  in  diesem  Fall
               auch mode=header.
       header  gibt  die Header-Daten auf stderr aus (setzt ifor-
               mat=header voraus).
       logfile protokolliert den Ablauf in  einer  Datei  mtcopy-
               NNNNN;  dabei  ist  NNNNN  die  Nummer  des Login-
               Prozesses.
       append  hängt die Daten an eine bereits bestehende  Datei
               an.  Diese  Option  ist nur erlaubt, wenn das Aus-
               gabegerät eine Datei auf Disk ist;  im        Fall  von
               Ausgabe   auf  Band  ist  ein  append-Modus  nicht
               möglich.

BEISPIELE

       mtcopy ifile=violet:/dev/dat ofile=/dev/dat format=mlle
               mode=header,interactive repeat=3
       kopiert 3 MLLE-Datensätze von DAT-Tape zu DAT-Tape.

       mtcopy ifile=/dev/dat format=mlle oformat=ascii | pg -esnc
       gibt den Inhalt einer MLLE-Datei auf TTY aus.

       mtcopy ifile=schroeder:/dev/dat ofile=charly:/dev/exabyte
               bsize=10240 format=header files=3 range=1,20:100
       kopiert Daten von DAT-Tape auf Exabyte;
       es  werden 3 Files zu einer zusammengefaßt, wobei nur der
       Header von File #1 übernommen wird;
       es werden  (außer  dem Header-Record)  jeweils  nur  die
       Records 20 ... 100 kopiert.
